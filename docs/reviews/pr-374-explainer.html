PR #374 Explainer: Reliable Long Message Delivery in tmux

Issue #373 fix for swallowed Enter after paste-buffer in ao send paths, including multi-chunk paste and busy render races.

What This PR Does

  • Hardens long/multiline tmux send flow where messages are delivered through paste-buffer.
  • Adds adaptive wait + Enter retry behavior with pane-change confirmation in core tmux helper.
  • Hardens runtime-tmux plugin with paste-settle detection and submission verification heuristics.
  • Fixes review feedback issues: baseline timing bug, unbounded delay, and timer-flaky tests.

Architecture and Design Choices

Core tmux helper (packages/core/src/tmux.ts)

  • Capture pane baseline right before first Enter (not before adaptive delay).
  • Cap adaptive delay with MAX_ADAPTIVE_PASTE_DELAY_MS = 15000.
  • Retry Enter with staggered delays when output does not progress.

Runtime tmux plugin (packages/plugins/runtime-tmux/src/index.ts)

  • Waits for paste rendering to settle to handle chunked paste markers.
  • Detects submission start via draft marker disappearance / prompt transition.
  • Retries Enter if submission not confirmed.
